# 《雨夜书房》spike

文档 §9 说的"一次性 spike"：3 个 AI 玩家 + 1 个 AI DM，纯命令行跑完一局，
验证三个最吓人的假设——**AI 肯不肯演/会不会自爆、推理行不行、延迟卡不卡**。

零第三方依赖，只用 Python 标准库 + 你的硅基流动 key。

## 文件

| 文件 | 作用 |
|---|---|
| `scenario.py` | 剧本数据（角色/秘密/线索/真相），无逻辑 |
| `visibility.py` | 可见性闸门雏形：算出某玩家被允许看到的信息 |
| `llm.py` | 用 `urllib` 直连硅基流动的最薄封装（零依赖） |
| `game.py` | 主循环：开场→自我介绍→搜证→讨论→投票→复盘 |
| `test_visibility.py` | 信息隔离自检（唯一必须确定性正确的部分） |
| `transcripts/` | 每局自动存档，供回放 |

## 跑起来

```powershell
# 1) 先跑不需要 key 的隔离自检
python test_visibility.py

# 2) 设置你的硅基流动 key（只在当前终端有效）
$env:SILICONFLOW_API_KEY = "你的key"

# 3) 跑整局（默认玩家和 DM 都用 DeepSeek-V3）
python game.py
```

想换模型（可选）：

```powershell
$env:PLAYER_MODEL = "deepseek-ai/DeepSeek-V3"
$env:DM_MODEL     = "deepseek-ai/DeepSeek-R1"   # DM 用更强的推理模型
```

## 关于 `think` / `say` / `action`

玩家每轮输出一个 JSON：私有 `think`（内心盘算 + 推理脚手架）、`action`（动作）、`target`（对象）、公开 `say`。
- **只有 `say` 进公开日志、被别人看到**；`think` 永不广播，只回流给该玩家自己当记忆。
- 控制台上 `🧠 …（私有·不广播）` 那行就是 `think`，**打出来纯粹是给你观察推理质量的**，对局里别人看不到它。
- 这样一来：发言更像真人（不带旁白/表情），心里又真在推理，且"内心独白被广播"这条泄密缝被堵死。

### 有边界的自主（动作菜单）

圆桌讨论阶段，玩家不再被强制逐个发言，而是**自己选动作**（自我介绍/投票阶段仍强制 `speak`）：

| action | 控制台标记 | 含义 |
|---|---|---|
| `speak` | `🗣️` | 正常陈述 |
| `silent` | `🤐` | 这轮不开口、观察 |
| `question` | `🗣️ …当众质问 X` | 点名质问某人；**被质问者立刻回应一次**（封顶一次，不套娃） |
| `push_vote` | `📢 …提议大家投 X` | 带节奏、号召投某人 |

> 关键：自主只发生在阶段【内】，阶段推进和硬上限仍由主循环掌控——这就是文档 §12 说的"**有边界的自主**"，既鲜活又不卡死。
> 想离线验证这套控制流（不花 API），可用 mock LLM 注入 `game.chat` 跑 `game.run()`（见提交历史里的临时冒烟脚本写法）。

## 看什么（spike 的验收）

- **尬不尬**：`say` 是不是像牌桌真人说话（口语、简短、无旁白）。
- **会不会演/自爆**：陈博（真凶）有没有全程撒谎、守住"整夜没出房"，赛后"凶手自爆检测"是否为 ✅。
- **推理行不行**：看 `🧠 think` 里的推理链顺不顺；三人投票多数票是否指向陈博，有没有被"林雅指纹""苏婉争吵"两条障眼法带偏。
- **卡不卡**：赛后小结里的平均每次调用耗时（think/say 会让每次输出更长，留意延迟变化）。
