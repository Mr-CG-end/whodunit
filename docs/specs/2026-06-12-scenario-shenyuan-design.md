# 难本一号《沈园夜宴》— 设计（Phase 1 子项目4）

> 上位设计见 [2026-06-05-whodunit-design.md](2026-06-05-whodunit-design.md) 修订 2026-06-12：eval 实测 WUYE 上各配置指标全持平，剧本太简单导致 eval 失去区分度；「第二个剧本」从 Phase 3 提前，作为恢复区分度的手段。WUYE 不动，继续当冒烟剧本。
>
> 剧本为**原创**（借用公共领域诡计手法），不检索/改编商业剧本——拿不到完整结构、有版权风险、且人桌本不贴合 InfoItem 体系，改编成本高于原创。

## 1. 范围

**做**：

1. **难本一号《沈园夜宴》**——5 AI 玩家、民国公馆题材、双骨架（强红鲱鱼 + 链式拼图），结构化为 `Scenario` 数据（新文件 `src/engine/scenario-shenyuan.ts`），引擎零改动。
2. **eval/CLI 选剧本**——`run.ts` / `play.ts` 加 `--scenario=wuye|shenyuan`（默认 wuye）。
3. **剧本结构不变量测试**——确定性、TDD（见 §6）。

**不做**：

- **难本二号（时间线诡计 + 顶罪/掩护结构）**——进 backlog；等一号 eval 数据证明「这种难法真能区分配置」再做，否则多写的本全白费。
- **引擎/阶段改动**——沿用 8 阶段 `PHASES`，与 WUYE 的 eval 数据同构可比：剧本是唯一变量。
- **golden-text 断言**——铁律；剧本质量靠结构不变量 + eval 指标 + 人工看 transcript。

## 2. 难度命题

eval 的核心区分指标是 **accuracyRate（投中真凶率）**。WUYE 的问题：公开线索单独就把陈博钉死，任何配置都接近全对。难本一号的设计目标反过来：

- **只看公开线索 ⇒ 投错**（红鲱鱼是公开线索下的唯一「合理」答案）；
- **拼合 directed 碎片 + 反直觉推理 ⇒ 投对**（链式拼图必须靠讨论交换私有信息）。

校准基准 = 现有默认配置：玩家 `DeepSeek-V4-Flash`、DM `DeepSeek-V4-Pro`（零代码改动）。**预期**：Flash 桌大概率被红鲱鱼带偏；讨论质量高（强模型/带 DM）的桌能拼出真凶——这正是区分度的来源。

## 3. 剧本骨架

**背景**：民国十六年，上海。实业家**沈仲鸿**六十寿宴当晚，深夜死于公馆「沈园」书房。死因：**乌头碱中毒**——毒下在他每晚睡前必饮的安神药酒里，药酒坛存于内院暖阁。

**5 位玩家**：

| 角色 | 身份 | 秘密（private） | 局中作用 |
|---|---|---|---|
| 顾曼珠 | 二姨太 | 当晚溜进书房偷地契准备私奔，见沈已昏迷，吓得逃走；妆匣里还藏着一包一念之差买下又没敢用的砒霜 | **强红鲱鱼**：动机+目击+物证三件套俱全 |
| 沈砚秋 | 大少爷，留洋归来 | **真凶**。挪用钱庄款项炒金亏空、伪造父亲签章的汇票；寿宴前夜被父亲对账识破、扬言送他去巡捕房。数日前从医生药箱偷得乌头碱，宴席间借「替父取陈酒」溜进暖阁下毒入坛 | 表面最干净：宴后陪客打牌至凌晨，「不在场证明」完美 |
| 白慕乔 | 家庭医生 | 多年前误诊害死过人，被沈拿捏才做私人医生（弱动机噪音） | **链碎片①毒源**持有者 |
| 程伯衡 | 老管家 | 多年虚报采买中饱私囊，怕查账查到自己头上 | **链碎片②机会**持有者 |
| 唐霜 | 世交之女、报馆记者 | 父亲当年破产自尽与沈仲鸿吞并有关，接近沈家本为挖丑闻（旧怨动机噪音） | **链碎片③动机**持有者 |

**线索发放**（公开 7 + 定向 4 + 秘密 5 + 真相 1 = 17 条 InfoItem）：

- **搜证1（表面层，搭红鲱鱼）**：
  - D1 公开｜法医报告：死因乌头碱中毒，毒在安神药酒里；沈每晚睡前必饮一盅，由仆人从暖阁酒坛温一壶送书房。
  - D2 公开｜仆人证词：宴散后见顾曼珠端参汤进过书房，出来时神色慌张。
  - D3 公开｜搜查：顾曼珠妆匣夹层搜出一包**砒霜**。
  - D4 公开｜律师透露：沈数日前约见律师拟改遗嘱，大幅削减顾曼珠名分与份额。
  - D5 定向→顾曼珠｜你进书房是取保险柜里的地契，当时沈已昏迷椅上，你吓得逃了——你没下毒，但没人会信。
- **搜证2（链条层）**：
  - D6 公开｜钱庄对账记录：寿宴前夜钱庄送来对账册，沈当夜在书房大发雷霆、摔了茶盏。
  - D7 公开｜暖阁勘查：药酒坛封蜡有新近开启痕迹，坛盖封布的系法与老仆习惯不同——宴会当晚有外人动过坛。
  - D11 公开｜牌局证词：宴散后大少爷与三位客人打牌至凌晨两点，期间未离席。（**不在场陷阱**，故意公开）
  - D8 定向→白慕乔｜你药箱里的乌头碱制剂上周短少；那几日药箱只离过一次手——在沈园出诊时留在大少爷院中的外间。
  - D9 定向→程伯衡｜宴间你守在内院廊下，亲眼见大少爷独自进过暖阁一趟，说是「替父亲取陈酒」；除他外无人进过内院。（同时完成对其他人的机会排除）
  - D10 定向→唐霜｜寿宴前你为访谈到公馆，在书房窗外听见沈与大少爷激烈争执，听清「汇票」「巡捕房」「逆子」。
- **真相（omniscient，复盘揭）**：凶手沈砚秋；三链闭环 + 红鲱鱼归位（砒霜系顾曼珠自购未用；牌局不在场真实但无效——毒在宴间已下入坛）。

**难度机关（三件）**：

1. **红鲱鱼层（防直觉投票）**：D2+D3+D4 把顾曼珠钉死；唯一破绽在公开线索内部——**D1 死因是乌头碱，D3 搜出的是砒霜，毒物不符**。锚定「砒霜+动机」直接投她即翻车。
2. **不在场陷阱（防排除法偷懒）**：D11 牌局证明真实但无效——下毒时间（宴间入坛）≠ 死亡时间（睡前饮酒）。因牌局排除大少爷即翻车。
3. **链式拼图（防单人破案）**：定罪沈砚秋必须 D8（毒源）+ D9（机会）+ D10（动机）三块 directed 碎片拼合，分别在三个**各怀心事、不愿多嘴**的玩家手里；公开线索单独看永远不够。

**公平性约束**：复盘时所有线索闭环、无悬空；全信息共享下真凶唯一可推（D9 同时排除他人机会，D1×D3 毒物不符洗白顾曼珠）。

## 4. 数据落点与既有机制的衔接

- `Scenario` 接口、`PHASES`、`item()` 模式照搬 `scenario.ts` 既有约定；新文件只装数据：`export const SHENYUAN: Scenario`。
- **aliases 按 3a 决策「宁漏不误伤」逐条审**：秘密的 alias 不得与任何公开/定向线索文本撞车（例：「汇票」出现在 D10 定向线索里，绝不能做凶手秘密的 alias，否则唐霜合法讨论即被误杀）。此约束写成确定性测试（§6）。
- 凶手秘密沿用 `tags: ["secret", "killer"]`（SELF_BURY 派生依赖它）；truth 沿用**无 aliases**约定（加了会误伤合法指认）。
- DMSpeaker 自动适配：`publicContext` 不含新剧本秘密/真相是架构保证，directed 仍只宣布「某人收到私下线索」。

## 5. eval/CLI 接入

- `src/eval/run.ts`、`src/play.ts`：`--scenario=wuye|shenyuan`，默认 `wuye`（现有脚本/文档零破坏）。剧本注册表就是一个 `Record<string, Scenario>`，不做 ScenarioLoader 文件解析（YAGNI，两个内置剧本而已）。
- 验证路径：`npm run eval -- --scenario=shenyuan`（K=3 起步）对照 WUYE 基线——**成功标准：shenyuan 的 accuracyRate 显著低于 WUYE（WUYE 近全对）**；若 Flash 桌照样一投一个准，说明红鲱鱼不够毒，回炉调剧本文本（不调引擎）。

## 6. 测试策略（确定性，TDD）

`src/engine/scenario-shenyuan.test.ts`：

- 结构不变量：`killer ∈ participants`；所有 private/directed 均有 owner；所有 `revealPhase ∈ phases`；omniscient 仅 truth 一条且 revealPhase=复盘。
- 链碎片不变量：D8/D9/D10 的 owner 互不相同、均非凶手、均非红鲱鱼本人（防止「碎片在凶手手里被永久隐瞒」破坏可解性）。
- 防误伤不变量：每条秘密 alias 不是任何其他 InfoItem 文本的子串（「宁漏不误伤」的机器化）。
- 现有 `visibility.test.ts` 的不变量对新剧本参数化跑一遍（VisibilityGate 对新剧本同样成立）。

LLM 部分照铁律不做文本断言：跑 eval 看结构指标 + 人工读 transcript 验「红鲱鱼是否够毒、链碎片是否被交换」。

## 7. 决策点

| | 决策 | 选择 | 状态 |
|---|---|---|---|
| A | 剧本来源 | 原创（借公共领域诡计手法），不检索/改编商业本 | 已确认 |
| B | 诡计骨架 | 强红鲱鱼 + 链式拼图；时间线+顶罪留难本二号进 backlog | 已确认 |
| C | 规模/题材 | 5 AI 玩家，民国公馆 | 已确认 |
| D | 阶段结构 | 沿用 8 阶段 PHASES，引擎零改动、与 WUYE 同构可比 | 已确认 |
| E | 难度校准 | 按默认 Flash 玩家档校准：公开线索⇒投错红鲱鱼，拼链⇒投对 | 已确认 |
| F | 多本节奏 | 今天只做一号；二号等一号 eval 验证区分度后启动 | 已确认 |
